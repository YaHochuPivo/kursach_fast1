# Как проверить логи сервера

## Где находятся логи

### 1. В консоли (где запущено приложение)
Логи выводятся в консоль, где вы запустили приложение:
- Если запускаете через IDE (IntelliJ IDEA, Eclipse), логи видны в окне "Run" или "Console"
- Если запускаете через командную строку (`mvn spring-boot:run` или `java -jar`), логи выводятся в терминал

### 2. В файле логов
Логи также сохраняются в файл: `logs/application.log` в корне проекта

## Как посмотреть логи

### В IDE (IntelliJ IDEA):
1. Найдите окно **"Run"** или **"Console"** внизу IDE
2. Там будут все логи приложения
3. Можно фильтровать по уровню (ERROR, WARN, INFO, DEBUG)

### В командной строке:
1. Если запускаете через `mvn spring-boot:run`, логи видны прямо в терминале
2. Если запускаете через `java -jar`, логи выводятся в консоль

### В файле:
**Важно:** Файл `logs/application.log` создается автоматически при первом запуске приложения. Если файла нет, значит приложение еще не запускалось после настройки логирования.

1. Запустите приложение (если еще не запущено)
2. После запуска откройте файл `logs/application.log` в любом текстовом редакторе
3. Или используйте команду: `Get-Content logs/application.log -Wait` (Windows PowerShell)

## Что искать в логах при ошибке ERR_INCOMPLETE_CHUNKED_ENCODING

Ищите строки, содержащие:
- `ERROR` - критические ошибки
- `Exception` - исключения
- `Ошибка при создании объявления` - ошибки создания объявления
- `Ошибка при загрузке` - ошибки загрузки данных
- `LazyInitializationException` - проблемы с lazy loading
- `SQLException` - ошибки базы данных

## Уровни логирования

- **ERROR** - критические ошибки (красные)
- **WARN** - предупреждения (желтые)
- **INFO** - информационные сообщения (белые)
- **DEBUG** - детальная отладочная информация (серые)
- **TRACE** - очень детальная информация (самый низкий уровень)

## Примеры полезных команд

### Просмотр последних 100 строк логов:
```bash
tail -n 100 logs/application.log
```

### Поиск ошибок в логах:
```bash
grep ERROR logs/application.log
```

### Мониторинг логов в реальном времени:
```bash
tail -f logs/application.log
```

### Windows PowerShell:
```powershell
Get-Content logs/application.log -Tail 100
Select-String -Path logs/application.log -Pattern "ERROR"
Get-Content logs/application.log -Wait
```
